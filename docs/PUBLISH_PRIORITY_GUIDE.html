FindMyMaid — Publish priority (iOS + Android)

App IDs: iOS bundle & Android package com.findmymaid.app
Website: https://www.findmymaid.online
Print to PDF: Use Ctrl+P → choose “Save as PDF” or “Microsoft Print to PDF”.

Tier 1 — Must do first (nothing ships without these)

# What Why
1 Paid accounts — Apple Developer Program + Google Play Console Both stores require enrolment before you can submit.
2 Expo / EAS — eas login, link project (eas init or first eas build), confirm eas whoami Release builds go through EAS from Windows or any OS.
3 Public backend — HTTPS API, DATABASE_URL, JWT_SECRET, prisma migrate deploy, Twilio TWILIO_* live, no OTP_DEV_BYPASS in prod Real SMS OTP and sync; LAN-only is not enough for store apps.
4 Set EXPO_PUBLIC_API_URL on EAS production Store builds will not reach your API without this.

Until Tier 1 is done, “publish” isn’t real.

Tier 2 — Required to submit to each store

# What Why
5 eas build --platform android --profile production (AAB) and iOS production build Store-ready binaries.
6 Play Console — app, package ID, privacy URL, listings, Data safety, rating, internal test Android gate.
7 App Store Connect — app, bundle ID, privacy URL, encryption, screenshots, App Privacy iOS gate.
8 eas submit -p android / eas submit -p ios (or manual upload) Upload to Play / TestFlight / review.
9 Reviewer notes — phone + real SMS OTP (Twilio) Reviewers must complete sign-in.

Tier 3 — Strongly recommended before “production”

# What Why
10 Website live (privacy, terms, support, pricing, contact) Policy URLs; avoid 404s.
11 Subscriptions: real IAP or hide scope (see docs/STORE_MONETIZATION.md) Demo billing may confuse review.
12 End-to-end device test — both roles, OTP, Find + API Catch URL / TLS / Twilio early.

Tier 4 — Quality (can follow v1)

# What Why
13 Profile photos via HTTPS URLs file:// won’t work on other devices.
14 Trim “Demo” UI copy if claiming production Cleaner review story.
15 Bump version / build per release Store hygiene.

One-line order of attack

Accounts → EAS login + project → deploy API + Twilio + prod env → EXPO_PUBLIC_API_URL → production EAS builds → Play + App Store Connect → internal testing → submit + reviewer OTP notes → Tier 3–4.
